Sphinx & Github pages

Sat, Feb 4, 2017 github

This tutorial shows how to use Github pages + Sphinx for publish nice looking documentation.

Install Sphinx and napoleon

pip install sphinx
pip install sphinxcontrib-napoleon

Configuration of the project in Sphinx

In the main folder of the project create a new folder with the name of docs.

mkdir docs
cd docs

The main folder must look like:

  • project_name
    • docs
    • module
    • build
    • README.txt
    • setup.py

Write the next command to start the sphinx configuration:

sphinx-quickstart

Add sources path for python documentation

Uncomment the line in the source/conf.py script

sys.path.insert(0, os.path.abspath('.'))

Change the argument ‘.’ to set the path of the source files, example:

sys.path.insert(0, os.path.abspath('../../<module_name>'))

Add classes/modules

For we need to modify the file index.rst to add links to the documentation of each module. The next one is a simple example:

Suppose that our library have 2 modules or classes called module_1 and module_2. We can add the link to the documentation of each module modifying the index.rst as

Contents:

.. toctree::
   :maxdepth: 2

   module_1
   module_2

For each module create a new .rst file with the name of each module, for example module_1.rst and module_2.rst

For each of this files write something like the bellow example for module_1

module_1 documentation!
======================================


.. automodule:: module_1
  :members:
  :undoc-members:
  :show-inheritance:

These files can also be autogenerated using sphinx-apidoc command

sphinx-apidoc -f -o docs/source

Where main_project_path is the the path of the main directory of the project

Function for compile and clean

Inside the doc directory put

make html

for complie and

make clean

for clean the documentation

Auto publication of the documentation

Create a .sh file with the next lines:

#!/usr/bin/env bash


# build the docs
cd docs
make clean
make html
cd ..

# commit and push
git add -A
git commit -m "pushing docs"
git push origin master

# switch branches and pull the data we want
git checkout gh-pages
rm -rf .
touch .nojekyll
git checkout master docs/build/html
mv ./docs/build/html/* ./
rm -rf ./docs
git add -A
git commit -m "publishing docs..."
git push origin gh-pages

# switch back
git checkout master

Put this script in the root of the project, run this script using the command:

sh autodoc.sh

Read the docs theme (working…):

Download the Read the Docs theme

pip install sphinx_rtd_theme

Add githubpages extension:

Open conf.py and in extension add the line

sphinx.ext.githubpages',

Then, the extensions will be as:

extensions = [
    'sphinx.ext.autodoc',
    'sphinx.ext.mathjax',
    'sphinx.ext.githubpages',
]

Import read the docs theme

import sphinx_rtd_theme
html_theme = "sphinx_rtd_theme"
html_theme_path = [sphinx_rtd_theme.get_html_theme_path()]

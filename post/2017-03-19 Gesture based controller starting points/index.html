Gesture based controller starting points

Mon, Mar 20, 2017 Python

General overview

The next diagram describe the general structure of the gesture based controller:

Sensory system

The sensory module is composed of 3 programs:

  • The mobile and wear version of the Nep Wear Ami: used to send the IMU data via sockets to a PC.
  • The ros_server.py program: Used to obtain the IMU information in the PC. This program also publish the IMU information in the topics ‘/wearami_acc’ in the case of the acceleration and ‘/wearami_acc’ in the case of gyroscope information (this are the names of the topics that I see in the current github version, but you can check if are the correct ones in your version, in the ICRA article it seems are with different names).

Perceptual system

The perceptual system consist in the gesture recognition module. Which main function is to subscribe to the ‘/wearami_acc’ topic and to infer the gesture that a person with the smartwatch perform using the GMM and GMR algortihms.
